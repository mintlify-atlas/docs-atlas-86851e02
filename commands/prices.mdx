---
title: prices
description: Manage plan pricing with subscriptions, one-time payments, free trials, and setup fees

The `prices` command provides complete control over plan pricing, including subscription management, one-time payments, free trials, setup fees, and price lifecycle management.

## Subcommands

### create

Create a new price for a plan.

```bash
memberstack prices create --plan-id <id> --name <name> --amount <amount> --type <type> [options]
```

#### Required Options

<ParamField path="--plan-id" type="string" required>
  Plan ID to add the price to (starting with `pln_`).
  
  ```bash
  --plan-id pln_premium
  ```
</ParamField>

<ParamField path="--name" type="string" required>
  Price name. Displayed to customers during checkout.
  
  ```bash
  --name "Premium Monthly"
  ```
</ParamField>

<ParamField path="--amount" type="number" required>
  Price amount in cents (or smallest currency unit).
  
  ```bash
  --amount 2900  # $29.00
  ```
</ParamField>

<ParamField path="--type" type="string" required>
  Price type. Accepts `SUBSCRIPTION` or `ONETIME`.
  
  ```bash
  --type SUBSCRIPTION
  ```
</ParamField>

#### Optional Parameters

<ParamField path="--currency" type="string" default="usd">
  Currency code (ISO 4217). Examples: `usd`, `eur`, `gbp`.
  
  ```bash
  --currency eur
  ```
</ParamField>

<ParamField path="--interval-type" type="string">
  Billing interval for subscriptions. Accepts `YEARLY`, `MONTHLY`, or `WEEKLY`.
  
  Required for `SUBSCRIPTION` type.
  
  ```bash
  --interval-type MONTHLY
  ```
</ParamField>

<ParamField path="--interval-count" type="number" default="1">
  Number of intervals between billings.
  
  Examples:
  - `1` = monthly (if interval-type is MONTHLY)
  - `3` = quarterly (if interval-type is MONTHLY)
  - `6` = semi-annually (if interval-type is MONTHLY)
  
  ```bash
  --interval-count 3  # Quarterly billing
  ```
</ParamField>

<ParamField path="--setup-fee-enabled" type="boolean">
  Enable a one-time setup fee.
  
  ```bash
  --setup-fee-enabled
  ```
</ParamField>

<ParamField path="--setup-fee-amount" type="number">
  Setup fee amount in cents.
  
  ```bash
  --setup-fee-amount 9900  # $99.00
  ```
</ParamField>

<ParamField path="--setup-fee-name" type="string">
  Setup fee name/description.
  
  ```bash
  --setup-fee-name "Onboarding Fee"
  ```
</ParamField>

<ParamField path="--free-trial-enabled" type="boolean">
  Enable free trial period.
  
  ```bash
  --free-trial-enabled
  ```
</ParamField>

<ParamField path="--free-trial-days" type="number">
  Free trial duration in days.
  
  ```bash
  --free-trial-days 14
  ```
</ParamField>

<ParamField path="--free-trial-requires-card" type="boolean">
  Require payment method during free trial signup.
  
  ```bash
  --free-trial-requires-card
  ```
</ParamField>

<ParamField path="--expiration-count" type="number">
  Number of expiration intervals.
  
  ```bash
  --expiration-count 12
  ```
</ParamField>

<ParamField path="--expiration-interval" type="string">
  Expiration interval unit. Accepts `MONTHS` or `DAYS`.
  
  ```bash
  --expiration-interval MONTHS
  ```
</ParamField>

<ParamField path="--cancel-at-period-end" type="boolean">
  Automatically cancel subscription at the end of the billing period.
  
  ```bash
  --cancel-at-period-end
  ```
</ParamField>

#### GraphQL Mutation

### update

Update an existing price.

```bash
memberstack prices update <priceId> [options]
```

<Warning>
  Updating active prices affects existing subscriptions. Consider creating a new price instead.
</Warning>

#### Arguments

<ParamField path="priceId" type="string" required>
  Price ID to update (starting with `pri_`).
  
  ```bash
  memberstack prices update pri_abc123xyz --amount 3900
  ```
</ParamField>

#### Options

<ParamField path="--name" type="string">
  Update price name.
  
  ```bash
  --name "Premium Plus Monthly"
  ```
</ParamField>

<ParamField path="--amount" type="number">
  Update price amount in cents.
  
  ```bash
  --amount 3900
  ```
</ParamField>

<ParamField path="--interval-count" type="number">

### activate

Reactivate a deactivated price.

```bash
memberstack prices activate <priceId>
```

#### Arguments

<ParamField path="priceId" type="string" required>
  Price ID to reactivate (starting with `pri_`).
  
  ```bash
  memberstack prices activate pri_abc123xyz
  ```
</ParamField>

#### GraphQL Mutation

```graphql
mutation($input: ReactivatePriceInput!) {
  reactivatePrice(input: $input) {
    id
    name
    status
    active
  }
}
```

<Note>
  Reactivating a price makes it available for new purchases but does not affect existing subscriptions.
</Note>

### deactivate

```bash
memberstack prices deactivate <priceId>
```

#### Arguments

<ParamField path="priceId" type="string" required>
  Price ID to deactivate (starting with `pri_`).
  
  ```bash
  memberstack prices deactivate pri_abc123xyz
  ```
</ParamField>

#### GraphQL Mutation

```graphql
mutation($input: DeactivatePriceInput!) {
  deactivatePrice(input: $input) {
    id
    name
    status
    active
  }
}
```

<Note>
  Deactivating a price prevents new purchases but does not affect existing subscriptions or members.
</Note>

## Common Workflows

### Creating Tiered Pricing

<Steps>
  <Step title="Create monthly prices for each tier">
    ```bash

    memberstack prices create \
      --plan-id pln_starter \
      --name "Starter Monthly" \
      --amount 900 \
      --type SUBSCRIPTION \
      --interval-type MONTHLY
    
    # Professional tier
    memberstack prices create \
      --plan-id pln_pro \
      --name "Professional Monthly" \
      --amount 2900 \
      --type SUBSCRIPTION \
      --interval-type MONTHLY
    
    # Enterprise tier
    memberstack prices create \
      --plan-id pln_enterprise \
      --name "Enterprise Monthly" \
      --amount 9900 \
      --type SUBSCRIPTION \
      --interval-type MONTHLY
    ```

  </Step>

  <Step title="Add annual options with discount">

    ```bash
    memberstack prices create \
    ```

  </Step>
</Steps>

### Adding Free Trials

```bash
# 14-day trial without requiring card
memberstack prices create \
```

```bash

memberstack prices create \
  --plan-id pln_premium \
  --name "Premium - Holiday Special" \
  --amount 1900 \
  --type SUBSCRIPTION \
  --interval-type MONTHLY \
```

```bash
# Sunset old pricing
memberstack prices deactivate pri_old_price

memberstack prices activate pri_old_price

# Permanently remove unused price
memberstack prices delete pri_unused
```

## Pricing Best Practices

### Amount Formatting

Always specify amounts in cents (or smallest currency unit):

- $9.99 = `--amount 999`
- $29.00 = `--amount 2900`
- $199.99 = `--amount 19999`
- €50.00 = `--amount 5000 --currency eur`

### Free Trials

<Tip>
  Use `--free-trial-requires-card` for higher conversion rates. Trials without card requirements have lower conversion but reduce friction.
</Tip>

### Setup Fees

Setup fees are charged once at the beginning of a subscription:

```bash
memberstack prices create \
  --plan-id pln_enterprise \
  --name "Enterprise Monthly" \
  --amount 29900 \
  --type SUBSCRIPTION \
  --interval-type MONTHLY \
  --setup-fee-enabled \
  --setup-fee-amount 99900 \
  --setup-fee-name "Onboarding & Configuration"
```

Total first charge: $299.00 + $999.00 = $1,298.00

Subsequent charges: $299.00/month

### Expiration

Use expiration for:
- Time-limited access (e.g., course access)
- Trial extensions
- Promotional periods

```bash
# 6-month access
memberstack prices create \
  --plan-id pln_course \
  --name "Course Access" \
  --amount 19900 \
  --type ONETIME \
  --expiration-count 6 \
  --expiration-interval MONTHS
```
