This guide will walk you through authenticating with Memberstack and running your first commands in just a few minutes.

## Before you begin

Make sure you have:

- **Installed the CLI** - See the [installation guide](/installation) if you haven't already
- **A Memberstack account** - Sign up at [memberstack.com](https://www.memberstack.com) if you don't have one
- **A terminal or command line interface** open and ready

## Step 1: Authenticate with OAuth

<Steps>

    ```bash
    memberstack auth login
    ```

  The OAuth flow uses a local server on `http://127.0.0.1` with a random available port to receive the authorization callback. This is completely secure and runs only on your machine.

## Step 2: Verify your identity

Now that you're authenticated, verify your identity and see which app you're connected to:

```bash
memberstack whoami
```

You'll see output like:

```
  App:   My Membership Site (app_1a2b3c4d5e6f)
  Status: active
  User:  john@example.com
```

This confirms:
- **App** - The name and ID of your Memberstack app
- **Status** - Whether the app is active
- **User** - The email address of your authenticated account

## Step 3: Run your first commands

Now you're ready to start managing your Memberstack account from the terminal!

### List all members

View all members in your app:

```bash
memberstack members list
```

This displays a formatted table of all members with their ID, email, creation date, and status.

### Create a new plan

Create a new membership plan:

```bash
memberstack plans create --name "Pro Plan" --description "Full access to all features"
```

Output:

```
вң” Plan created successfully.

  id:          pln_1a2b3c4d5e6f
  name:        Pro Plan
  description: Full access to all features
  createdAt:   2026-03-01T10:30:00Z
```

### Export data to JSON

Get raw JSON output for scripting and automation:

```bash
memberstack members list --json
```

This outputs raw JSON to stdout, perfect for piping into other tools:

```json
[
  {
    "id": "mem_abc123",
    "auth": {
      "email": "user@example.com"
    },
    "createdAt": "2026-01-15T08:30:00Z"
  }
]
```

## Understanding environment modes

The CLI supports both sandbox and live environments. By default, all commands run in **sandbox mode**.

### Sandbox mode (default)

```bash
memberstack members list
```

Or explicitly:

```bash
memberstack members list --sandbox
```

### Live mode

To work with your production data, use the `--live` flag:

```bash
memberstack members list --live
```

Or set the mode explicitly:

```bash
memberstack members list --mode live
```

<Warning>
  Be careful when using `--live` mode. Commands in live mode affect your production data and are irreversible.
</Warning>

You can also set the default mode using an environment variable:

```bash
export MEMBERSTACK_MODE=live
memberstack members list
```

## Check authentication status

At any time, you can check your authentication status:

```bash
memberstack auth status
```

This shows:
- Login status
- App ID
- Token expiration time
- Refresh token availability
- Token validity

Example output:

```
  Status:       Logged in
  App ID:       app_1a2b3c4d5e6f
  Expires in:   2h 30m
  Refresh:      Available
  Token:        Valid
```

## Global options

The CLI supports several global options that work with any command:

| Option | Description |
|--------|-------------|
| `-j, --json` | Output raw JSON instead of formatted tables |
| `-q, --quiet` | Suppress banner and non-essential output |
| `--no-color` | Disable color output |
| `--mode <mode>` | Set environment mode (`sandbox` or `live`) |
| `--live` | Shorthand for `--mode live` |
| `--sandbox` | Shorthand for `--mode sandbox` |

### Examples

<CodeGroup>
```bash JSON output
memberstack members list --json
```

```bash Quiet mode
memberstack members list --quiet
```

```bash No color
memberstack members list --no-color
```

```bash Live mode
memberstack members list --live
```
</CodeGroup>

## Logging out

When you're done, you can log out to remove your stored tokens:

```bash
memberstack auth logout
```

This revokes your refresh token and deletes the local `~/.memberstack/auth.json` file.
